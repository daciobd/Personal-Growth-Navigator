Perfeito — agora vamos montar um **dashboard real de growth**, do jeito que founders e PMs usam no dia a dia.

Isso aqui é **plug-and-play no Mixpanel ou PostHog**.

---

# 🧠 1. VISÃO GERAL DO DASHBOARD

Você vai criar **4 blocos principais**:

1. 🚀 Ativação (primeira vitória)
2. 🔁 Retenção (volta do usuário)
3. ⚡ Engajamento (uso real)
4. 💰 Sinais de valor (indicadores fortes)

---

# 🚀 2. ATIVAÇÃO (O MAIS IMPORTANTE)

## 🔽 FUNIL PRINCIPAL

```txt
onboarding_started
→ onboarding_problem_selected
→ plan_generated
→ practice_started
→ practice_completed
```

---

## 📊 Gráfico: Funnel

No Mixpanel/PostHog:

* tipo: **Funnel**
* conversão por step
* janela: 24h

---

## 🎯 O que olhar

* onde maior drop acontece
* tempo entre steps

---

## 🚨 Diagnóstico rápido

| Problema                        | Interpretação      |
| ------------------------------- | ------------------ |
| Drop alto no plan_generated     | plano não convence |
| Drop alto no practice_started   | CTA fraco          |
| Drop alto no practice_completed | prática ruim       |

---

# 🔁 3. RETENÇÃO (COHORT)

## 📊 Gráfico: Retention

Evento base:

```txt
practice_completed
```

Evento de retorno:

```txt
app_opened
```

---

## Configuração

* tipo: **Retention → Cohort**
* intervalo: diário
* período: últimos 30 dias

---

## 🎯 Meta saudável

| Dia | Meta |
| --- | ---- |
| D1  | 30%+ |
| D3  | 15%+ |
| D7  | 10%+ |

---

## 🧠 Insight poderoso

Se D1 for baixo:

👉 problema NÃO é retenção
👉 problema é ativação fraca
